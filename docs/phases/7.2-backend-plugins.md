# Phase 7.2: Backend Plugins

> Custom resource backends via the plugin system.

---

## Goal

Enable plugins to provide custom resource backends (Kubernetes, cloud providers, etc.) that integrate seamlessly with dtx orchestration.

---

## User Value

```
- Community backends without forking dtx
- Kubernetes, AWS ECS, Nomad support via plugins
- Consistent orchestration across all backends
```

---

## Dependencies

```
Phase 1.1 (Resource Trait) - Backends implement Resource
Phase 1.2 (EventBus v2) - Lifecycle events
Phase 7.1 (Plugin Loader) - Plugin infrastructure
```

### 7.2.1 BackendPlugin Trait Refinement (2h)

**File:** `crates/dtx-plugin/src/backend.rs`

```rust
use async_trait::async_trait;
use dtx_core::{
    resource::{Resource, ResourceId, ResourceKind, ResourceConfig},
    events::EventBus,
    context::Context,
    error::Result,
};
use std::sync::Arc;

/// A plugin that provides a resource backend.
///
/// Backend plugins create resources of a specific kind that integrate
/// with dtx orchestration.
#[async_trait]
pub trait BackendPlugin: Send + Sync {
    /// The resource kind this backend handles.
    fn kind(&self) -> ResourceKind;

    /// Human-readable name of the backend.
    fn name(&self) -> &str;

    /// Backend version.
    fn version(&self) -> &str;

    /// Create a resource from configuration.
    ///
    /// The config is parsed from the YAML resource definition.
    /// The event_bus is shared and should be used for lifecycle events.
    fn create(
        &self,
        id: ResourceId,
        config: serde_json::Value,
        event_bus: Arc<EventBus>,
    ) -> Result<Box<dyn Resource>>;

    /// Validate configuration before creating resource.
    ///
    /// Called during config parsing to provide early feedback.
    fn validate_config(&self, config: &serde_json::Value) -> Result<()> {
        let _ = config;
        Ok(())
    }

    /// Check if this backend is available.
    ///
    /// Called during startup to verify dependencies (e.g., kubectl exists).
    async fn health_check(&self) -> Result<BackendHealth> {
        Ok(BackendHealth::Healthy)
    }

    /// Get schema for the resource configuration.
    ///
    /// Used for documentation and validation.
    fn config_schema(&self) -> Option<schemars::schema::RootSchema> {
        None
    }

    /// Backend-specific capabilities.
    fn capabilities(&self) -> BackendCapabilities {
        BackendCapabilities::default()
    }
}

/// Health status of a backend.
#[derive(Clone, Debug)]
pub enum BackendHealth {
    Healthy,
    Degraded { reason: String },
    Unavailable { reason: String },
}

impl BackendHealth {
    pub fn is_healthy(&self) -> bool {
        matches!(self, Self::Healthy)
    }
}

/// Capabilities a backend supports.
#[derive(Clone, Debug, Default)]
pub struct BackendCapabilities {
    /// Supports health checks.
    pub health_checks: bool,
    /// Supports log streaming.
    pub logs: bool,
    /// Supports exec into resource.
    pub exec: bool,
    /// Supports port forwarding.
    pub port_forward: bool,
    /// Supports metrics.
    pub metrics: bool,
    /// Supports restart without recreate.
    pub hot_restart: bool,
}

impl BackendCapabilities {
    pub fn full() -> Self {
        Self {
            health_checks: true,
            logs: true,
            exec: true,
            port_forward: true,
            metrics: true,
            hot_restart: true,
        }
    }

    pub fn with_health_checks(mut self) -> Self {
        self.health_checks = true;
        self
    }

    pub fn with_logs(mut self) -> Self {
        self.logs = true;
        self
    }
}
```

### 7.2.2 Backend Registry (2h)

**File:** `crates/dtx-plugin/src/backend_registry.rs`

```rust
use std::collections::HashMap;
use std::sync::Arc;

use dtx_core::resource::ResourceKind;

use super::backend::{BackendPlugin, BackendHealth, BackendCapabilities};

/// Registry of backend plugins.
///
/// Provides lookup by resource kind and manages backend lifecycle.
pub struct BackendRegistry {
    backends: HashMap<ResourceKind, Arc<dyn BackendPlugin>>,
}

impl BackendRegistry {
    pub fn new() -> Self {
        Self {
            backends: HashMap::new(),
        }
    }

    /// Register a backend plugin.
    pub fn register(&mut self, backend: Arc<dyn BackendPlugin>) -> &mut Self {
        self.backends.insert(backend.kind(), backend);
        self
    }

    /// Get backend for a resource kind.
    pub fn get(&self, kind: &ResourceKind) -> Option<Arc<dyn BackendPlugin>> {
        self.backends.get(kind).cloned()
    }

    /// Check if a backend is registered for this kind.
    pub fn has_backend(&self, kind: &ResourceKind) -> bool {
        self.backends.contains_key(kind)
    }

    /// List all registered backends.
    pub fn list(&self) -> Vec<BackendInfo> {
        self.backends
            .values()
            .map(|b| BackendInfo {
                kind: b.kind(),
                name: b.name().to_string(),
                version: b.version().to_string(),
                capabilities: b.capabilities(),
            })
            .collect()
    }

    /// Check health of all backends.
    pub async fn health_check_all(&self) -> HashMap<ResourceKind, BackendHealth> {
        let mut results = HashMap::new();
        for (kind, backend) in &self.backends {
            let health = backend.health_check().await.unwrap_or_else(|e| {
                BackendHealth::Unavailable { reason: e.to_string() }
            });
            results.insert(*kind, health);
        }
        results
    }
}

impl Default for BackendRegistry {
    fn default() -> Self {
        Self::new()
    }
}

/// Information about a registered backend.
#[derive(Clone, Debug)]
pub struct BackendInfo {
    pub kind: ResourceKind,
    pub name: String,
    pub version: String,
    pub capabilities: BackendCapabilities,
}
```

### 7.2.3 Backend Configuration Schema (2h)

**File:** `crates/dtx-plugin/src/backend_config.rs`

```rust
use schemars::JsonSchema;
use serde::{Deserialize, Serialize};

/// Base configuration that all backend resources share.
#[derive(Clone, Debug, Serialize, Deserialize, JsonSchema)]
pub struct BackendResourceConfig {
    /// Resource kind (matches the backend).
    pub kind: String,

    /// Environment variables.
    #[serde(default)]
    pub environment: std::collections::HashMap<String, String>,

    /// Dependencies on other resources.
    #[serde(default)]
    pub depends_on: Vec<DependencyConfig>,

    /// Restart policy.
    #[serde(default)]
    pub restart: RestartPolicyConfig,

    /// Health check configuration.
    #[serde(default)]
    pub health_check: Option<HealthCheckConfig>,

    /// Labels for grouping.
    #[serde(default)]
    pub labels: std::collections::HashMap<String, String>,

    /// Backend-specific configuration.
    #[serde(flatten)]
    pub backend_config: serde_json::Value,
}

/// Dependency configuration.
#[derive(Clone, Debug, Serialize, Deserialize, JsonSchema)]
pub struct DependencyConfig {
    pub resource: String,
    #[serde(default)]
    pub condition: DependencyCondition,
}

/// Condition for dependency satisfaction.
#[derive(Clone, Debug, Default, Serialize, Deserialize, JsonSchema)]
#[serde(rename_all = "snake_case")]
pub enum DependencyCondition {
    #[default]
    Started,
    Healthy,
    Completed,
}

/// Restart policy configuration.
#[derive(Clone, Debug, Default, Serialize, Deserialize, JsonSchema)]
#[serde(rename_all = "snake_case")]
pub enum RestartPolicyConfig {
    #[default]
    No,
    Always,
    OnFailure,
}

/// Health check configuration.
#[derive(Clone, Debug, Serialize, Deserialize, JsonSchema)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum HealthCheckConfig {
    Exec { command: Vec<String> },
    Http { path: String, port: u16 },
    Tcp { port: u16 },
    Grpc { port: u16 },
}

/// Parse backend config from YAML value.
pub fn parse_backend_config<T: serde::de::DeserializeOwned>(
    value: serde_json::Value,
) -> Result<T, serde_json::Error> {
    serde_json::from_value(value)
}

/// Merge base config with backend-specific config.
pub fn merge_configs(
    base: &BackendResourceConfig,
    backend_defaults: serde_json::Value,
) -> serde_json::Value {
    let mut merged = backend_defaults;
    if let serde_json::Value::Object(ref mut map) = merged {
        // Apply base environment
        if !base.environment.is_empty() {
            map.insert(
                "environment".to_string(),
                serde_json::to_value(&base.environment).unwrap(),
            );
        }
    }
    merged
}
```

### 7.2.4 Resource Factory (2h)

**File:** `crates/dtx-plugin/src/resource_factory.rs`

```rust
use std::sync::Arc;

use dtx_core::{
    resource::{Resource, ResourceId, ResourceKind},
    events::EventBus,
    error::{Error, Result},
};

use super::backend_registry::BackendRegistry;

/// Factory for creating resources using registered backends.
pub struct ResourceFactory {
    backends: Arc<BackendRegistry>,
    event_bus: Arc<EventBus>,
}

impl ResourceFactory {
    pub fn new(backends: Arc<BackendRegistry>, event_bus: Arc<EventBus>) -> Self {
        Self { backends, event_bus }
    }

    /// Create a resource from configuration.
    pub fn create(
        &self,
        id: ResourceId,
        kind: ResourceKind,
        config: serde_json::Value,
    ) -> Result<Box<dyn Resource>> {
        let backend = self.backends.get(&kind).ok_or_else(|| {
            Error::config(format!("no backend registered for kind: {}", kind))
        })?;

        // Validate config first
        backend.validate_config(&config)?;

        // Create the resource
        backend.create(id, config, Arc::clone(&self.event_bus))
    }

    /// Create multiple resources from a configuration map.
    pub fn create_many(
        &self,
        configs: Vec<(ResourceId, ResourceKind, serde_json::Value)>,
    ) -> Result<Vec<Box<dyn Resource>>> {
        configs
            .into_iter()
            .map(|(id, kind, config)| self.create(id, kind, config))
            .collect()
    }

    /// Check if a resource kind is supported.
    pub fn supports_kind(&self, kind: &ResourceKind) -> bool {
        self.backends.has_backend(kind)
    }

    /// List supported resource kinds.
    pub fn supported_kinds(&self) -> Vec<ResourceKind> {
        self.backends.list().into_iter().map(|b| b.kind).collect()
    }
}
```

### 7.2.5 Built-in Process Backend as Plugin (3h)

use async_trait::async_trait;
use std::sync::Arc;

use dtx_core::{
    resource::{Resource, ResourceId, ResourceKind},
    events::EventBus,
    error::Result,
};
use dtx_plugin::{BackendPlugin, BackendCapabilities, BackendHealth};

use super::process::Process;
use super::config::ProcessConfig;

/// The built-in process backend.
///
/// This is implemented as a plugin to demonstrate the pattern and
/// allow it to be replaced if needed.
pub struct ProcessBackendPlugin;

#[async_trait]
impl BackendPlugin for ProcessBackendPlugin {
    fn kind(&self) -> ResourceKind {
        ResourceKind::Process
    }

    fn name(&self) -> &str {
        "process"
    }

    fn version(&self) -> &str {
        env!("CARGO_PKG_VERSION")
    }

    fn create(
        &self,
        id: ResourceId,
        config: serde_json::Value,
        event_bus: Arc<EventBus>,
    ) -> Result<Box<dyn Resource>> {
        let process_config: ProcessConfig = serde_json::from_value(config)?;
        Ok(Box::new(Process::new(id, process_config, event_bus)))
    }

    fn validate_config(&self, config: &serde_json::Value) -> Result<()> {
        // Parse to validate structure
        let _: ProcessConfig = serde_json::from_value(config.clone())?;
        Ok(())
    }

    async fn health_check(&self) -> Result<BackendHealth> {
        // Process backend is always available
        Ok(BackendHealth::Healthy)
    }

    fn capabilities(&self) -> BackendCapabilities {
        BackendCapabilities {
            health_checks: true,
            logs: true,
            exec: false,
            port_forward: false,
            metrics: false,
            hot_restart: true,
        }
    }

    fn config_schema(&self) -> Option<schemars::schema::RootSchema> {
        Some(schemars::schema_for!(ProcessConfig))
    }
}

/// Register the process backend with a registry.
pub fn register_process_backend(registry: &mut dtx_plugin::BackendRegistry) {
    registry.register(Arc::new(ProcessBackendPlugin));
}

### 7.2.6 Example: Kubernetes Backend Plugin (3h)

**File:** `crates/dtx-plugin/examples/kubernetes/src/lib.rs`

```rust
//! Example Kubernetes backend plugin.
//!
//! This demonstrates how to create a backend plugin for an external
//! orchestration system.

use async_trait::async_trait;
use schemars::JsonSchema;
use serde::{Deserialize, Serialize};
use std::sync::Arc;

use dtx_core::{
    resource::{Resource, ResourceId, ResourceKind, ResourceState, HealthStatus},
    events::EventBus,
    context::Context,
    error::Result,
};
use dtx_plugin::{
    dtx_plugin, Plugin, BackendPlugin, BackendCapabilities, BackendHealth,
};

// --- Plugin Registration ---

pub struct KubernetesPlugin;

impl Plugin for KubernetesPlugin {
    fn name(&self) -> &str { "kubernetes" }
    fn version(&self) -> &str { env!("CARGO_PKG_VERSION") }
    fn api_version(&self) -> u32 { 2 }

    fn backends(&self) -> Vec<Box<dyn BackendPlugin>> {
        vec![Box::new(KubernetesBackend::new())]
    }
}

impl KubernetesPlugin {
    pub fn new() -> Self { Self }
}

dtx_plugin!(KubernetesPlugin);

// --- Backend Implementation ---

pub struct KubernetesBackend {
    namespace: String,
}

impl KubernetesBackend {
    pub fn new() -> Self {
        Self {
            namespace: std::env::var("DTX_K8S_NAMESPACE")
                .unwrap_or_else(|_| "default".to_string()),
        }
    }
}

#[async_trait]
impl BackendPlugin for KubernetesBackend {
    fn kind(&self) -> ResourceKind {
        ResourceKind::Custom(100) // Kubernetes = 100
    }

    fn name(&self) -> &str { "kubernetes" }
    fn version(&self) -> &str { "0.1.0" }

    fn create(
        &self,
        id: ResourceId,
        config: serde_json::Value,
        event_bus: Arc<EventBus>,
    ) -> Result<Box<dyn Resource>> {
        let k8s_config: KubernetesResourceConfig = serde_json::from_value(config)?;
        Ok(Box::new(KubernetesResource::new(
            id,
            k8s_config,
            self.namespace.clone(),
            event_bus,
        )))
    }

    fn validate_config(&self, config: &serde_json::Value) -> Result<()> {
        let _: KubernetesResourceConfig = serde_json::from_value(config.clone())?;
        Ok(())
    }

    async fn health_check(&self) -> Result<BackendHealth> {
        // Check if kubectl is available
        match tokio::process::Command::new("kubectl")
            .arg("version")
            .arg("--client")
            .output()
            .await
        {
            Ok(output) if output.status.success() => Ok(BackendHealth::Healthy),
            Ok(_) => Ok(BackendHealth::Degraded {
                reason: "kubectl not configured".to_string(),
            }),
            Err(e) => Ok(BackendHealth::Unavailable {
                reason: format!("kubectl not found: {}", e),
            }),
        }
    }

    fn capabilities(&self) -> BackendCapabilities {
        BackendCapabilities {
            health_checks: true,
            logs: true,
            exec: true,
            port_forward: true,
            metrics: true,
            hot_restart: false,
        }
    }

    fn config_schema(&self) -> Option<schemars::schema::RootSchema> {
        Some(schemars::schema_for!(KubernetesResourceConfig))
    }
}

// --- Configuration ---

#[derive(Clone, Debug, Serialize, Deserialize, JsonSchema)]
pub struct KubernetesResourceConfig {
    /// Container image.
    pub image: String,

    /// Command to run.
    #[serde(default)]
    pub command: Vec<String>,

    /// Arguments to command.
    #[serde(default)]
    pub args: Vec<String>,

    /// Environment variables.
    #[serde(default)]
    pub env: std::collections::HashMap<String, String>,

    /// Ports to expose.
    #[serde(default)]
    pub ports: Vec<u16>,

    /// Number of replicas.
    #[serde(default = "default_replicas")]
    pub replicas: u32,

    /// Resource requests/limits.
    #[serde(default)]
    pub resources: Option<ResourceRequirements>,
}

fn default_replicas() -> u32 { 1 }

#[derive(Clone, Debug, Serialize, Deserialize, JsonSchema)]
pub struct ResourceRequirements {
    pub cpu: Option<String>,
    pub memory: Option<String>,
}

// --- Resource Implementation ---

pub struct KubernetesResource {
    id: ResourceId,
    config: KubernetesResourceConfig,
    namespace: String,
    state: ResourceState,
    event_bus: Arc<EventBus>,
}

impl KubernetesResource {
    pub fn new(
        id: ResourceId,
        config: KubernetesResourceConfig,
        namespace: String,
        event_bus: Arc<EventBus>,
    ) -> Self {
        Self {
            id,
            config,
            namespace,
            state: ResourceState::Pending,
            event_bus,
        }
    }

    fn deployment_name(&self) -> String {
        format!("dtx-{}", self.id.as_str())
    }
}

#[async_trait::async_trait]
impl Resource for KubernetesResource {
    fn id(&self) -> &ResourceId { &self.id }
    fn kind(&self) -> ResourceKind { ResourceKind::Custom(100) }
    fn state(&self) -> &ResourceState { &self.state }

    async fn start(&mut self, _ctx: &Context) -> Result<()> {
        self.state = ResourceState::Starting { started_at: chrono::Utc::now() };

        // Apply kubernetes manifest
        let manifest = self.generate_deployment_yaml();

        let mut child = tokio::process::Command::new("kubectl")
            .args(["apply", "-f", "-", "-n", &self.namespace])
            .stdin(std::process::Stdio::piped())
            .spawn()?;

        if let Some(stdin) = child.stdin.as_mut() {
            use tokio::io::AsyncWriteExt;
            stdin.write_all(manifest.as_bytes()).await?;
        }

        let status = child.wait().await?;
        if !status.success() {
            return Err(dtx_core::error::Error::backend("kubectl apply failed"));
        }

        self.state = ResourceState::Running {
            pid: None,
            started_at: chrono::Utc::now(),
        };

        Ok(())
    }

    async fn stop(&mut self, _ctx: &Context) -> Result<()> {
        let status = tokio::process::Command::new("kubectl")
            .args(["delete", "deployment", &self.deployment_name(), "-n", &self.namespace])
            .status()
            .await?;

        if !status.success() {
            return Err(dtx_core::error::Error::backend("kubectl delete failed"));
        }

        self.state = ResourceState::Stopped {
            exit_code: Some(0),
            started_at: chrono::Utc::now(),
            stopped_at: chrono::Utc::now(),
        };

        Ok(())
    }

    async fn health(&self) -> HealthStatus {
        let output = tokio::process::Command::new("kubectl")
            .args([
                "get", "deployment", &self.deployment_name(),
                "-n", &self.namespace,
                "-o", "jsonpath={.status.readyReplicas}",
            ])
            .output()
            .await;

        match output {
            Ok(o) if o.status.success() => {
                let ready = String::from_utf8_lossy(&o.stdout);
                if ready.parse::<u32>().unwrap_or(0) >= self.config.replicas {
                    HealthStatus::Healthy
                } else {
                    HealthStatus::Unhealthy {
                        reason: "not all replicas ready".to_string(),
                    }
                }
            }
            _ => HealthStatus::Unknown,
        }
    }

    fn as_any(&self) -> &dyn std::any::Any { self }
    fn as_any_mut(&mut self) -> &mut dyn std::any::Any { self }
}

impl KubernetesResource {
    fn generate_deployment_yaml(&self) -> String {
        format!(
            r#"
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {name}
spec:
  replicas: {replicas}
  selector:
    matchLabels:
      app: {name}
  template:
    metadata:
      labels:
        app: {name}
    spec:
      containers:
      - name: main
        image: {image}
        ports:
        {ports}
"#,
            name = self.deployment_name(),
            replicas = self.config.replicas,
            image = self.config.image,
            ports = self.config.ports
                .iter()
                .map(|p| format!("        - containerPort: {}", p))
                .collect::<Vec<_>>()
                .join("\n"),
        )
    }
}
```
